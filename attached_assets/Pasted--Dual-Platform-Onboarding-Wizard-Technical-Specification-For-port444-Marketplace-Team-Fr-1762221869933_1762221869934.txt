## üéØ Executive Summary

This document describes the **Dual-Platform Onboarding Wizard** that enables seamless account creation across Based Creators and port444 marketplace using a single wallet signature. Users complete one form and automatically get accounts on both platforms, creating a powerful cross-platform creator ecosystem.

---

## üåü Overview

### What is Dual-Platform Onboarding?

The Dual-Platform Onboarding Wizard allows creators to:
- ‚úÖ **Sign up once** with their Web3 wallet
- ‚úÖ **Choose their platform** (just Based Creators, just port444, or BOTH)
- ‚úÖ **One unified profile** shared across both platforms
- ‚úÖ **Automatic account sync** via secure API integration

### Key Benefits

**For Users:**
- Single wallet authentication across both platforms
- One profile, two powerful creator networks
- No duplicate data entry
- Verified wallet-based identity

**For Platforms:**
- Shared user growth
- Cross-platform referrals
- Verified user base
- API-driven integration

### OAuth Implementation

For each social platform, implement standard OAuth 2.0 flow:

#### GitHub OAuth
```typescript
const connectGitHub = async () => {
  // 1. Redirect to GitHub OAuth
  const clientId = process.env.GITHUB_CLIENT_ID;
  const redirectUri = `${window.location.origin}/auth/github/callback`;
  const scope = 'read:user user:email';
  
  window.location.href = 
    `https://github.com/login/oauth/authorize?client_id=${clientId}&redirect_uri=${redirectUri}&scope=${scope}`;
};

// 2. Handle callback
app.get('/auth/github/callback', async (req, res) => {
  const { code } = req.query;
  
  // Exchange code for token
  const tokenResponse = await fetch('https://github.com/login/oauth/access_token', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      client_id: process.env.GITHUB_CLIENT_ID,
      client_secret: process.env.GITHUB_CLIENT_SECRET,
      code
    })
  });
  
  // Get user info
  const userResponse = await fetch('https://api.github.com/user', {
    headers: { Authorization: `Bearer ${accessToken}` }
  });
  
  const { login, html_url } = await userResponse.json();
  
  // Store in session/state
  setFormData({ ...formData, githubUrl: html_url });
});
```

#### Twitter OAuth (Similar pattern)
- Use Twitter OAuth 2.0 PKCE flow
- Scope: `users.read tweet.read`
- Store username and profile URL

#### Farcaster Auth
- Use Farcaster Auth Kit or Sign-In With Farcaster
- Verify FID (Farcaster ID)
- Store username and FID

#### Zora & Base Profile
- Use ENS resolution for Base names
- Verify on-chain profile ownership
- Link to profile pages

```sql
-- Users table
CREATE TABLE users (

  wallet_address VARCHAR(42) UNIQUE NOT NULL,

  email VARCHAR(255),

  base_profile VARCHAR(255),
  
  -- port444 sync tracking

  port444_sync_status VARCHAR(20), -- 'synced', 'pending', 'failed'
  port444_synced_at TIMESTAMP,
  
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()

-- Cross-platform mapping
CREATE TABLE cross_platform_mappings (
  id SERIAL PRIMARY KEY,
  wallet_address VARCHAR(42) NOT NULL,
  based_creators_user_id UUID,
  port444_user_id UUID,
  sync_status VARCHAR(20),
  last_sync_attempt TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);
```

```sql
  -- Based Creators sync
  based_creators_user_id UUID,
  based_creators_sync_status VARCHAR(20),
  based_creators_synced_at TIMESTAMP,
  
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Create indexes
CREATE INDEX idx_builders_wallet ON builders(wallet_address);
CREATE INDEX idx_builders_categories ON builders USING GIN(categories);
CREATE INDEX idx_builders_skills ON builders USING GIN(skills);
```

## üîê Security Considerations

### Wallet Verification
- All authentication based on wallet signatures
- No password storage needed
- Wallet address is primary key

### OAuth Security
- State parameter for CSRF protection
- PKCE flow for Twitter OAuth 2.0
- Secure token storage (httpOnly cookies)
- Token refresh handling

### API Security
- API keys stored in environment variables
- Request signing for cross-platform sync
- Rate limiting on external endpoints
- Idempotent operations for retry safety

### Data Privacy
- Users control which platforms to join
- Clear consent for data sharing
- OAuth tokens never shared between platforms
- Right to disconnect accounts
